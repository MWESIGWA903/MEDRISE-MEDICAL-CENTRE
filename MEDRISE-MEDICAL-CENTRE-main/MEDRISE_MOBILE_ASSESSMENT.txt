MedRise Mobile is an Expo (React Native) staff-facing application enabling
patient queue management, vitals recording, ward rounds, inpatient care,
attendance tracking, and patient lookups. It targets Android and iOS devices
and is built with the new React Native architecture (newArchEnabled: true).

The app is functional and well-structured, with proper authentication, loading
states, and pull-to-refresh on all list screens. It uses the shared
@workspace/api-client-react library for most data fetching, ensuring
consistency with the web ERP.

[MS-01] TOKEN STORED IN UNENCRYPTED ASYNCSTORAGE — HIGH
──────────────────────────────────────────────────────────
  File: artifacts/medrise-mobile/lib/AuthContext.tsx
  Issue: Authentication tokens are stored in React Native AsyncStorage, which
         is unencrypted and accessible to any native code on a rooted device.
  Impact: Token theft on compromised devices.
  Fix: Use expo-secure-store (SecureStore) for token storage instead.
  Code change:
    // Replace:
    await AsyncStorage.setItem("token", token);
    // With:
    import * as SecureStore from "expo-secure-store";
    await SecureStore.setItemAsync("token", token);

[MS-02] NO CERTIFICATE PINNING — MEDIUM
─────────────────────────────────────────
  Issue: No SSL certificate pinning is configured. Man-in-the-middle attacks
         are theoretically possible on untrusted networks.
  Fix: For production healthcare apps, consider react-native-ssl-pinning.
       For a staff-only internal app, this is low urgency.

[MS-03] SENSITIVE DATA IN LOGS — MEDIUM
─────────────────────────────────────────
  Issue: Fixed: console.warn statements now log network error codes.
         Ensure no patient names/diagnoses are logged in production.
  Status: ✅ Fixed during this audit (was silent catch, now logs status codes only)
